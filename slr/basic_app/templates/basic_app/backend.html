Welcome to the Back End Section.

In order to make a successful pull request, you must be able to set up the backend, according to the following methods.

It would help for you to look into the given ideas, before you proceed to work on the back end.

  • Consider looking into the django docs, on how to create a new application.
  • Check into how you should connect views, templates, and urls in Django.
  • A good practice would be to set up your own Django website, before you work on this one, in case you do not know much about Django.
  • Look into template inheritance in order to understand how basic_app's web pages are working.

The tasks below are the ones which are expected to be accomplished, in order for a successful pull request.

  • Creating a new application using django commands.
  • Create views, and templates for the new application.
  • Connect the new application to slr, using settings.py
  • Either use the base template or make your own, and create a simple web page, with a title, a subtitle, and a lorem Ipsum paragraph.
  • Be sure to connect the application, such that it can be accessed when the django runs the server.
  • (Optional) Add in a navbar-link into the navbar on the index page, to link to your application's display page.
